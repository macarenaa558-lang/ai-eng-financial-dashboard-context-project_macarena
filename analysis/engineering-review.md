# Engineering Review

## Good practices

1. Clear separation between frontend and backend.
2. Strong typing with TypeScript and Pydantic.
3. Modular React components.
4. Loading and error handling in the UI.
5. REST API organized with specialized endpoints.

## Improvement opportunities

1. Split routes.py into smaller modules.
2. Move business logic into service classes/functions.
3. Replace mock data with a persistence layer.
4. Restrict CORS configuration for production.
5. Reduce the responsibilities of routes.py to improve maintainability.
