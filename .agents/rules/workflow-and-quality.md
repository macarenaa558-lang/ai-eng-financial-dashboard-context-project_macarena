# Workflow and Quality Rules

- Review `AGENTS.md`, `.agents/rules`, `.agents/skills`, and `memory-bank` before modifying the project.
- Make small, focused changes.
- Do not redesign the application or add features unless explicitly requested.
- Run the relevant tests after modifying frontend or backend logic.
- Run frontend linting when changing TypeScript or React files.
- Verify that Docker Compose still starts both services after structural changes.
- Keep documentation synchronized with important architectural changes.
- Use one focused Git commit for each completed phase or meaningful unit of work.
- Write clear commit messages describing the completed change.
